Introduction

"Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design. Built by experienced developers, it takes care of much of the hassle of web development, so you can focus on writing your app without needing to reinvent the wheel. It’s free and open source." -djangoproject.com

Personally I'm using Django as the web framework for my revamped personal site. I wanted a way to dynamically load content to the web page, and to be able to make changes to the content of the site without redeploying. Django offers the ability to generalize your HTML so that you can populate fields in your HTML with data from a database. It's essentially the web standard of removing hard coded variables, and I love it!

Assumed Prior Knowledge

Django doesn't assume too much prior knowledge, but a solid understanding of the following can greatly ease your development process :

  • Python (duh) : This mainly comes into play when writing your Views, or logic
  • HTML : Relevant when writing advanced templates / utilizing the Template Language
  • Site Hosting : For if you want to show off your Django site to the world
MTV Architecture

Django essentially follows the standard MVC (Model, View, Controller) framework, which features separate components to control :

  • The Model : The source of truth data representing the application
  • The View : The version of model data displayed to the user
  • The Controller : Entity handling interaction between user, view, and data

but with a slightly different spin.

"In our interpretation of MVC, the “view” describes the data that gets presented to the user. It’s not necessarily how the data looks, but which data is presented. The view describes which data you see, not how you see it."

They instead propose the MTV architecture, for "Model", "Template", and "View". This makes much more sense in the context of Django, since you directly work with a model, which is fed into a template, which is rendered with a view. Interested to learn more? Check out their explanation at their FAQ.

Template Language

Django has a language for their templates, or HTML, which you can use to render variable input from your views. For example, when rendering a template, you can pass in context from your view. Your context is essentially a dictionary of variable name mappings, and you can access those variables in your HTML using the Django Template Language.

This allows you to do conditional rendering! What if you have an error message, and you want to custom render a page to display that error? Use the template language :

{% if error_message %}<&p>Sorry, we got an error : {{ error_message }}<&/p>

Or you have some object you're passing in, and you want to render one of it's attributes? Use the template language

<&p>Thanks so much {{ user.firstName }} for using our service!<&/p>
